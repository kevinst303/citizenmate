# Plan 06-03: Tiered Stripe Checkout API & Webhook Handling

<objective>
Refactor the existing Stripe integration to support recurring subscriptions for the Pro and Premium tiers, handling checkout session creation and webhooks robustly.
</objective>

## Tasks

1. **Tiered Stripe Checkout API**
   - Update `src/app/api/checkout/route.ts` to accept `tier` ('pro', 'premium') and `interval` ('month', 'year').
   - Map these to corresponding Stripe Price IDs from environment variables.
   - Set the Stripe checkout session `mode` to `'subscription'`.

2. **Stripe Webhook Handling**
   - Update `src/app/api/webhooks/stripe/route.ts` to correctly process `customer.subscription.created`, `customer.subscription.updated`, and `customer.subscription.deleted`.
   - Implement logic to map the active subscription tier back to the user's `profile` in Supabase (granting/revoking access).
   - Ensure idempotency remains intact using the existing `processed_webhook_events` table.

## Verification
- Test checkout endpoint with a mocked tier and interval.
- Simulate Stripe webhooks (`stripe trigger customer.subscription.created`) and verify `profiles` table updates correctly.
