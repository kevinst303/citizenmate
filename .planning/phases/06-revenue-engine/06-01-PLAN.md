# Plan 06-01: Onboarding Flow & Global Upgrade Modal

<objective>
Implement the mandatory test-date-anchored onboarding flow for new users and establish the global UpgradeModal component structure for handling tier upgrades (Pro/Premium).
</objective>

## Tasks

1. **Test-Date-Anchored Onboarding Flow**
   - Create `src/app/onboarding/page.tsx`. This page should guide new users to set their goal test date and current knowledge level.
   - Update `src/components/auth/auth-context.tsx` (or equivalent layout) to redirect users to `/onboarding` if they lack a test date. (Provide a skip or "I don't know yet" option for the date to avoid pure dead ends, but the flow itself is mandatory).

2. **Global Upgrade Modal Component & State**
   - Create `src/lib/store/useUpgradeModal.ts` (using Zustand or similar) to manage the global state of the upgrade modal (open/close, trigger source).
   - Implement `src/components/global/upgrade-modal.tsx` with a premium aesthetic comparing Pro and Premium tiers (use placeholders for exact prices for now: `NEXT_PUBLIC_STRIPE_PRO_MONTHLY_PRICE_ID` and `NEXT_PUBLIC_STRIPE_PREMIUM_MONTHLY_PRICE_ID`).

## Verification
- Sign up as a new user and ensure redirection to `/onboarding` occurs.
- Verify test date is correctly set in the profile after completion.
- Verify the Upgrade Modal can be triggered and visually matches the premium design language.
