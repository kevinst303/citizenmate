# Plan 06-02: Six Distinct Upgrade Triggers

<objective>
Integrate the 6 strategic upgrade triggers across the application to drive free users to the paywall, leveraging the global UpgradeModal built in 06-01.
</objective>

## Tasks

1. **Dashboard Banner**: Add a persistent, dismissible "Upgrade to Premium" banner for free users on the main dashboard.
2. **Mock Test Limit**: Modify the mock test initialization logic. Allow 1 free test completion. Subsequent attempts should trigger the `UpgradeModal`.
3. **AI Tutor Daily Limit**: Add a rate-limit tracker for free users in `chat-context.tsx` or the relevant API. Show the paywall when exceeded.
4. **Advanced Readiness Analytics**: Obfuscate or lock the detailed topic breakdown component (`readiness-analytics.tsx` or similar) for free users with an upgrade CTA overlay.
5. **Study Mode Premium Content**: Lock specific study guides, translations, or advanced modules behind the paywall in the study mode interface.
6. **SRS Engine Review Limit**: Enforce a daily review limit for free users in the Spaced Repetition System logic; trigger paywall when the limit is reached.

## Verification
- Trigger all 6 upgrade prompts as a free user and ensure they open the `UpgradeModal`.
