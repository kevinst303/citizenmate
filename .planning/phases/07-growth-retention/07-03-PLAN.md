# Plan 07-03: Email Notifications

<objective>
Implement automated email notifications using Resend and Vercel Cron jobs to re-engage users (3-day inactivity), celebrate milestones (100 questions answered), and notify of successful referrals.
</objective>

## Tasks

1. **Email Templates and Resend Setup**
   <read_first>
   - package.json (to check for resend and react-email dependencies)
   </read_first>
   <action>
   Install `resend` and `@react-email/components` if not present.
   Create `src/lib/email/resend.ts` to export a configured Resend client using `process.env.RESEND_API_KEY`.
   Create React Email components in `src/components/emails/`: `InactivityEmail.tsx`, `MilestoneEmail.tsx`, `ReferralEmail.tsx`.
   Ensure each template includes a standardized unsubscribe link mapped to an unsubscribe route.
   </action>
   <acceptance_criteria>
   - `src/lib/email/resend.ts` exports the `resend` client.
   - Three email component files exist and use `@react-email/components`.
   </acceptance_criteria>

2. **Cron Job API Endpoint**
   <read_first>
   - src/app/api/cron/emails/route.ts
   </read_first>
   <action>
   Create `src/app/api/cron/emails/route.ts` as a GET handler.
   Implement secure access by checking the `Authorization` header against `process.env.CRON_SECRET`.
   Add logic to query Supabase for users who:
   - Haven't logged in for 3 days and haven't received the inactivity email recently.
   - Have exactly 100 questions answered and haven't received the milestone email.
   Loop through results and send emails via Resend.
   Update `vercel.json` to define the cron schedule for `/api/cron/emails` (e.g., daily at 00:00).
   </action>
   <acceptance_criteria>
   - `route.ts` exists and checks `CRON_SECRET`.
   - `vercel.json` contains a `"crons"` array with the correct path.
   </acceptance_criteria>

## Verification
- Hit the `/api/cron/emails` endpoint locally (mocking the `CRON_SECRET`) and verify the DB query executes.
- Ensure the React Email templates render without errors.
