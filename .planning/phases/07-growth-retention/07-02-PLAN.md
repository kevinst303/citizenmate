# Plan 07-02: Referral Program "Help a Mate"

<objective>
Implement the referral system to allow users to share `/?ref=userId` links, tracking referrers in Supabase and extending `premium_expires_at` by 7 days for both parties upon successful signup.
</objective>

## Tasks

1. **Schema and Database Triggers**
   <read_first>
   - supabase/migrations (to see existing structure)
   </read_first>
   <action>
   Create a new migration file `supabase/migrations/20260503000000_referrals.sql`.
   Add a `referred_by` column (UUID, referencing `profiles.id`) to the `profiles` table.
   Create a Supabase Database Trigger (or use the existing webhook handler) to extend `premium_expires_at` by 7 days for the new user and the user matching `referred_by` when a new profile is created with a `referred_by` value.
   Ensure abuse prevention by limiting successful referrals per user to 5 (add a `referral_count` column to track this).
   </action>
   <acceptance_criteria>
   - Migration SQL file exists and contains `ALTER TABLE profiles ADD COLUMN referred_by UUID REFERENCES profiles(id)`.
   - Migration SQL file contains logic to extend `premium_expires_at`.
   </acceptance_criteria>

2. **Cookie Tracking and Client Logic**
   <read_first>
   - src/lib/referrals.ts
   - src/app/layout.tsx (or equivalent global layout/middleware)
   </read_first>
   <action>
   Create or update `src/lib/referrals.ts` to export a function that parses the URL for `?ref=` and saves it to a cookie `citizenmate_ref`.
   Ensure this function is called on the landing page or global layout.
   Update the signup flow to read the `citizenmate_ref` cookie and pass it to the Supabase auth/profile creation process.
   </action>
   <acceptance_criteria>
   - `src/lib/referrals.ts` handles reading and writing the referral cookie.
   - The cookie `citizenmate_ref` is created when visiting `/?ref=uuid`.
   </acceptance_criteria>

## Verification
- Visit the app with `/?ref=test-uuid`.
- Verify the cookie is set.
- Mock a signup and verify the database migration logic correctly updates `referred_by` and extends the `premium_expires_at` date.
