## Executive Summary

Since the v1.1 milestone audit on 2026-05-05 (which rated GROW-01 "unsatisfied — zero UI integration"), the i18n system has undergone a **significant transformation**. The core infrastructure, dictionary pipeline, and component wiring are now functional. However, **5 critical-to-medium integration gaps** prevent non-English users from having a seamless experience:

1. **Arabic RTL broken** — No `dir="rtl"`, no RTL CSS
2. **28+ hardcoded links** — Every navigation triggers 307 redirect
3. **Stripe strips locale** — Users return to wrong language after payment
4. **NEXT_LOCALE cookie never set** — Language preference not persisted
5. **Blog links without locale** — Breaks non-English blog browsing

**Verdict: GROW-01 upgraded from unsatisfied → partial.** The foundation is solid but integration wiring has critical gaps.

## Current State: Language Switching & Translations

### What Works (End-to-End Verified)

1. **Middleware locale routing** — Accept-Language detection via Negotiator + @formatjs/intl-localematcher. Cookie override. Missing-locale redirect. Auth gating preserved. Referral cookie preserved. ✅
2. **I18nProvider → useT() → 29 components** — Provider receives locale from `params.lang`. Dynamic import of dictionary JSON. Graceful fallback on load failure. All client components inside `[lang]` have translations. ✅
3. **5 dictionaries, 674+ keys, 0% drift** — en.json (source), es.json, hi.json, zh.json, ar.json all fully translated. Landing section translated in all languages. ✅
4. **LanguageSwitcher URL routing** — Extracts locale from pathname, replaces segment, router.push(). Native language names in dropdown. Rendered in navbar (desktop + mobile). ✅
5. **Navbar locale-aware links** — `getUrl(path)` prepends `/${lang}`. All nav links use it. ✅
6. **Blog translation DB pipeline** — `getPublishedPosts(locale)`, `getPostBySlug(locale, slug)` join blog_translations. `generateStaticParams()` generates all (lang, slug) pairs. ✅
7. **Admin blog translation management** — CRUD API at `/api/admin/blog/translations`. Locale tabs in editor. Green dot indicator for filled translations. ✅
8. **Translation automation** — DeepL `scripts/translate.ts` for new keys. `scripts/validate-i18n.ts` checks keys, sections, tokens. `--check-only`, `--diff`, `--warn-only` modes. ✅
9. **Sitemap locale-aware** — English blog posts at `/blog/{slug}`, non-English at `/{locale}/blog/{slug}`. ✅
10. **Bilingual study toggle** — Separate from UI locale. en/zh/both via StudyContext. Persisted to localStorage + synced to Supabase. ✅
